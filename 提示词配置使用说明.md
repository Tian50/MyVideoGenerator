# 提示词配置使用说明

## 概述

已将项目中所有AI提示词集中到 `prompts_config.py` 文件中，便于统一管理和修改。

## 已修改的文件

以下文件已更新为使用统一的提示词配置：

1. `translator.py` - 翻译模块
2. `title_generator.py` - 标题生成模块
3. `blogger_info.py` - 博主信息查询模块
4. `highlight_detector.py` - 高光检测模块
5. `content_filter.py` - 内容过滤模块
6. `cover_generator.py` - 封面生成模块（火山引擎）
7. `local_cover_generator.py` - 本地封面生成模块

## 配置文件结构

`prompts_config.py` 包含以下类：

### 1. TranslatorPrompts - 翻译提示词
- `get_translation_prompt(target_lang="zh")` - 翻译字幕
- `get_correction_prompt()` - 检查翻译质量

### 2. TitleGeneratorPrompts - 标题生成提示词
- `get_title_generation_prompt()` - 生成视频标题

### 3. BloggerInfoPrompts - 博主信息提示词
- `get_blogger_info_prompt()` - 查询博主信息

### 4. HighlightDetectorPrompts - 高光检测提示词
- `get_highlight_detection_prompt(num=3)` - 检测高光片段

### 5. ContentFilterPrompts - 内容过滤提示词
- `get_content_filter_prompt()` - 检测违规内容

### 6. CoverGeneratorPrompts - 封面生成提示词
- `get_cover_design_prompt()` - 封面设计（火山引擎）
- `get_title_split_prompt()` - 标题分段（DeepSeek）

### 7. PromptManager - 提示词管理器
提供统一的访问接口，所有模块都通过这个类获取提示词。

## 如何修改提示词

### 方法1：直接编辑配置文件
1. 打开 `prompts_config.py`
2. 找到对应的提示词类
3. 修改返回的字符串内容
4. 保存文件

### 示例：修改标题生成提示词
```python
class TitleGeneratorPrompts:
    @staticmethod
    def get_title_generation_prompt():
        # 修改这里的提示词内容
        return '''{title_ref}{meta_desc}
下面的是视频字幕文本，请为这个视频起 **20 个**吸引人的标题...
（这里添加你的自定义要求）'''
```

### 方法2：添加新的提示词
如果需要添加新的提示词：
1. 在 `prompts_config.py` 中添加新的类
2. 在 `PromptManager` 中添加对应的方法

## 使用示例

### 在代码中使用提示词
```python
from prompts_config import PromptManager

# 获取翻译提示词
translation_prompt = PromptManager.get_translation_prompt("zh")

# 获取标题生成提示词（带参数）
title_prompt = PromptManager.get_title_generation_prompt().format(
    title_ref="视频原标题：测试\n",
    meta_desc="视频元数据：博主：张三"
)

# 获取博主信息提示词
blogger_prompt = PromptManager.get_blogger_info_prompt().format(
    author_name="John Smith"
)
```

### 查看所有提示词
运行 `python prompts_config_example.py` 可以查看所有提示词的内容。

## 优势

1. **集中管理**：所有提示词在一个文件中，便于查找和修改
2. **一致性**：确保整个项目使用相同的提示词风格
3. **可维护性**：修改提示词时只需修改一个文件
4. **可扩展性**：轻松添加新的提示词类别
5. **文档化**：每个提示词都有清晰的注释和说明

## 注意事项

1. 修改提示词后，所有使用该提示词的模块都会自动生效
2. 提示词中的占位符（如 `{title_ref}`、`{author_name}`）需要在代码中通过 `.format()` 方法填充
3. 建议在修改前备份原提示词，以便需要时恢复
4. 修改后建议运行测试确保功能正常
